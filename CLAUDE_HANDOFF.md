# Claude Handoff: 有肌訓練預約系統

## 專案目標

這是一個給「有肌訓練 / YUGYM Training」使用的健身房預約系統第一版原型。

主要用途：

- 用類似 Google 行事曆的週曆畫面查看預約。
- 管理教練課、友善教練課、小班團體課、自主訓練。
- 支援固定預約與票券堂數規則。
- 給健身房老闆、櫃台、教練快速查看場地容量。

## 目前主要檔案

- `index.html`：目前的頁面原型，包含 HTML、CSS、JavaScript。
- `BOOKING_SYSTEM_V1.md`：第一版功能規格。
- `AGENTS.md`：專案協作指引。
- `assets/yugym-logo.png`：原始品牌圖。
- `assets/yugym-logo-sidebar.png`：從品牌圖裁出的側邊欄 logo。
- `assets/yugym-logo-card.png`：備用裁切圖。

## 目前已完成的功能

### 1. Google 行事曆式週曆

- 一週七天欄位。
- 左側時間軸。
- 每個時段顯示容量狀態。
- 點擊時段會帶入右側新增預約表單。

### 2. 預約類型

目前包含：

- 教練課 1v1
- 教練課 1v2
- 友善教練課 1v1
- 友善教練課 1v2
- 小班團體課
- 自主訓練

### 3. 容量規則

一般容量：

- 同一時段最多 3 組。
- 教練課、友善教練課、自主訓練都會占用一般容量。
- 1v1 算 1 組。
- 1v2 也算 1 組。

小班團體課：

- 同一時段最多 1 組。
- 每組最多 4 人。
- 不占用一般容量 3 組。
- 可以和其他一般預約同時存在。

友善教練課：

- 只能在平日 09:00 到 18:00 預約。

### 4. 固定預約與票券規則

右側表單已加入：

- 每週重複預約開關。
- 固定頻率：一週一次 / 一週兩次。
- 有效票券堂數。
- 一週兩次時，可選第二次日期與第二次時間。

目前邏輯：

- 有 3 堂票券 + 一週一次 = 建立 3 週固定預約。
- 有 3 堂票券 + 一週兩次 = 建立 3 次固定預約，約 2 週完成。
- 建立前會逐筆檢查容量與規則。

### 5. 品牌

頁面已套用：

- 深綠色品牌側欄。
- 金色品牌細節。
- 側邊欄 logo：`assets/yugym-logo-sidebar.png`
- 頁面標題：有肌訓練預約系統。

### 6. 固定欄位

- 右側新增預約表單會跟著畫面固定。
- 左側今日狀態也會跟著畫面固定。

## 建議下一步

### 優先順序 1：把原型拆成正式專案

目前所有東西都在 `index.html`，適合原型。

下一步可改成：

- `src/` 結構
- React / Vue / Svelte 任一框架
- 或繼續維持純 HTML/CSS/JS

如果只是要快速試用，可以先不要拆。

### 優先順序 2：新增會員票券資料

目前票券堂數是手動在表單選。

下一步應該加入會員資料：

- 會員姓名
- 電話
- 方案類型
- 剩餘堂數
- 到期日
- 是否可預約

新增預約時，系統應該自動讀會員剩餘堂數。

### 優先順序 3：固定預約檢視

目前固定預約建立後，週曆只顯示第 1 週，其餘週次存在資料中。

下一步可加入：

- 下一週 / 上一週切換真的顯示不同 weekOffset。
- 固定預約群組列表。
- 可取消整組固定預約。
- 可取消其中一堂。

### 優先順序 4：資料持久化

目前資料存在前端記憶體，重新整理會消失。

下一步可選：

- localStorage：最快，適合測試。
- Google Sheet：適合老闆初期管理。
- Supabase / Firebase：適合做正式系統。
- 後端資料庫：適合完整產品。
