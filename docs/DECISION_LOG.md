# 重要決策紀錄

這份文件記錄已經討論過、會影響系統設計的決策。

## 2026/05/25

### 後台以電腦版為主

決策：

- 有肌訓練後台系統主要給老闆、店長、櫃台在電腦使用。
- 教練或會員手機版之後另外製作。

原因：

- 後台資料密度高，電腦版比較適合管理預約、會員、票券與營運摘要。
- 手機端需要更簡化，不適合直接塞完整後台。

### GitHub 同步程式碼，Supabase 同步資料

決策：

- GitHub 用來同步程式碼。
- Supabase 用來同步會員、預約、票券與營運資料。

原因：

- GitHub 不適合放真實會員資料。
- 瀏覽器本機資料無法可靠跨電腦同步。

### 展示版保留假資料

決策：

- 展示版使用假會員與假預約。
- 真實會員資料不得寫進 `js/app.js` 或上傳 GitHub。

原因：

- 保護會員隱私。
- 方便給朋友、工程師或外部人員看系統，不暴露真實資料。

### 分期提醒優先於續約提醒

決策：

- 同一張預約小卡不應同時顯示分期與續約。
- 如果該會員還有下一期付款未開放堂數，優先顯示分期。
- 只有同類票券全部用完且沒有後續分期時，才顯示續約。

原因：

- 分期代表同一份合約還沒走完，不需要重新簽約。
- 續約代表要重新簽一份合約。

### 友善教練課限制離峰

決策：

- 友善教練課只在平日 09:00 到 18:00 可預約。
- 新增預約時，非離峰時段不顯示友善教練課。

原因：

- 避免櫃台誤排。
- 符合實際營運規則。

### 同教練同時段不可重複預約

決策：

- 同一個教練同一時段不能有兩張教練課預約。

原因：

- 防止實務排課衝突。

### 小班團課獨立容量

決策：

- 一般預約同時段最多 3 組。
- 小班團課同時段只能有一組。
- 團課不與一般預約容量互斥。

原因：

- 團課與一般預約的管理邏輯不同。
### 2026-05-25 簽到點數區分

- 團體課簽到不發自主訓練點數，避免團課出席造成額外自主訓練儲值。
- 友善教練課簽到發「友善自主」，用途限定平日離峰時段，與一般自主訓練分開計算。
- 預約小卡儲存若沒有修改內容，直接關閉；只有有改動時才要求確認。
