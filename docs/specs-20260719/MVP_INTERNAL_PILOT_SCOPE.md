# YUGYM 櫃台與員工內部測試 MVP

## 目標

讓櫃台與員工能在不依賴正式會員端的情況下，安全完成一天的基本工作，並與舊系統雙軌核對。

## 本階段必須完成

### 櫃台

- 登入後只看到：今日櫃台、預約、會員、票券・收款。
- 查看當日全部預約及場地占用。
- 建立、修改、取消預約；正確扣堂、退堂或標記待補課。
- 代會員簽到；重複簽到不重複發點。
- 搜尋會員並查看基本資料、票券、預約與付款狀態。
- 建立收款及查看待處理購買／分期項目。
- 不可進入員工薪資、人資、完整財務、營運分析及系統設定。

### 一般員工／教練

- 查看今日課程與自己的行事曆。
- 查看自己的班表及完成上下班打卡。
- 提交補打卡申請。
- 查看自己的薪資試算與明細，不能查看他人薪資。
- 只能修改授權範圍內的個人資料與自己的課程紀錄。

### 主管

- 除一般員工功能外，只增加排班管理。
- 可查看、新增及調整全體班表。
- 不可查看員工薪資、人資、保險、財務或系統設定。

### 管理員（測試支援）

- 建立與停用測試帳號。
- 修正測試資料、處理補打卡與必要設定。
- 可同時具備主管身份；導覽需同時顯示「管理員」和「主管」。

## 暫不阻擋內部測試

- LINE／LIFF 會員登入。
- 會員全面自助預約與舊會員批次綁定。
- 完整員工合約版本、福利模板與個人覆寫管理頁。
- 完整票券規則設定器、場地規則設定器。
- 電子發票、支出、完整營運分析及進階報表。

## 開始內部測試的門檻

1. 測試 Supabase migration 與前端版本一致。
2. 管理員、櫃台、一般教練、主管四組測試帳號可登入。
3. 完成「建立預約 → 扣堂 → 修改 → 簽到 → 贈點 → 取消 → 退堂／不退堂」流程。
4. 櫃台與主管直接開啟未授權頁面會被阻擋，API／RLS 越權測試也失敗。
5. 員工打卡、補打卡、班表與個人薪資唯讀流程通過。
6. JavaScript 無語法錯誤，主要頁面沒有阻斷操作的錯誤。
7. 備妥每日對帳表與回到舊系統的方式。

## 7 天雙軌測試

每天結束核對：

- 預約新增、修改、取消與簽到數。
- 票券扣除、退回、贈點及補課狀態。
- 班表、打卡與補打卡差異。
- 教練有效堂數及預估堂獎。
- 權限錯誤、重複資料與操作失敗紀錄。

連續 7 天核心數字一致，才進入會員小規模測試。
