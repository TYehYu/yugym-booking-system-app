# YUGYM OS 上線與回復 SOP

## 分階段啟用

1. 內部 7 天：管理員、櫃台、教練雙軌操作；每天核對預約、扣退堂與簽到。
2. 會員小組 7 天：10–20 位會員從 LINE 官方帳號進入，觀察登入與舊資料連結。
3. 全會員：連續 7 天每日差異為零後，才把官方帳號圖文選單全面切換。
4. 舊系統唯讀：保留至少一個完整結算週期，再決定停用。

## 每日核對

- 當日新增、取消、簽到預約數。
- 票券扣除、退回、簽到贈點總額。
- 待審會員連結、核准與拒絕數。
- 異常／越權／RPC 錯誤紀錄。

## 停止條件

出現重複扣堂、錯誤綁定會員、越權讀取、批次無法對帳或無法登入達 15 分鐘，立即停止擴大使用。

## 回復

1. 將 LINE 圖文選單入口切回舊系統，YUGYM OS 暫停新寫入。
2. 保留事故期間 audit/ticket logs，不刪資料。
3. 以發布前備份和 migration 版本判定資料庫回復方式；禁止直接手改正式資料。
4. 匯出事故期間的有效交易，修復後重新套用並逐筆核對。
5. 完成原因、影響會員、補救與重新上線簽核。
