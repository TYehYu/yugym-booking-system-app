# 管理端與櫃台工作區規則

## 導覽顯示

### 櫃台帳號

只顯示每日營運需要的工作區：

- 今日櫃台
- 預約
- 會員
- 票券・收款

不顯示員工薪資、人資制度、營運分析、全域財務、權限及系統設定。

### 管理員帳號

除每日營運工作區外，導覽列增加「管理員」：

- 員工管理、出勤管理
- 財務總覽、營運分析
- 人資、薪資、勞健保、特休與工作規則
- 課程方案、動作資料庫、權限及系統設定

### 主管身份

`is_supervisor=true` 時導覽列增加「主管」。主管目前唯一職責為排班管理：

- 可查看全體員工班表。
- 可新增、修改及調整班表。
- 不因此取得員工薪資、人資、保險、財務、營運分析、權限或系統設定。
- 若主管同時需要管理功能，帳號必須另外具備管理員身份。

### 身兼二職

管理員帳號同時具有主管身份時，導覽列同時顯示「管理員」與「主管」，兩者不互相取代。

## 權限原則

- 導覽隱藏只是操作介面，不能當成資料安全機制。
- 前端 `canOpenPage()` 阻擋殘留連結或直接切頁。
- Supabase RLS／RPC 必須再次驗證實際角色與職務。
- 櫃台不得透過 `employees` 全表取得底薪、銀行、身分證、保險與完整薪資欄位。
- 櫃台若需要員工姓名、班表或授課資料，應改讀只含必要欄位的安全 view／RPC。

## 後續資料庫工作

1. 建立櫃台可讀的 `staff_directory` 安全 view，只提供 ID、顯示名、職務、狀態及排課必要欄位。
2. 將 `employees` 敏感欄位的直接讀取限縮為管理員及本人。
3. 將薪資、人資、保險 RPC 限縮為管理員；主管只取得明確授權的排班 RPC。
4. 以管理員、主管、管理員兼主管、櫃台四組帳號執行導覽與越權 UAT。
