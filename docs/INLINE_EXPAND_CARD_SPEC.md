# YUGYM OS · Inline Expand Card 設計規範

> 來源：`YUGYM_OS_Inline_Expand_Card_設計規範.docx`（2026-07，使用者提供）。本檔為文字版轉錄，供版控與開發參考。
> **狀態：規劃中，逐階段實作。** 進度見文末。

## 設計理念

YUGYM OS **不使用傳統 Modal（彈出視窗）** 進行預約、修改或簽到。所有高頻操作皆在行事曆中完成，
讓使用者始終保留目前的瀏覽情境，不中斷工作流程。

## 一、核心互動原則

- **Hover**：卡片 Fish-eye 放大，顯示完整資訊。
- **Click**：卡片原地展開（Inline Expand）。
- **完成操作後**：卡片收回原始尺寸。
- 全系統（新增、修改、簽到、取消）皆採用相同互動方式。

## 二、新增預約

1. 滑鼠移至空白時段，顯示淡化的「＋新增預約」。
2. 點擊後**不跳出視窗**，而是在該時段直接展開建立卡片。
3. 日期、時間自動帶入，不需重新選擇。
4. 僅需選擇：課程 / 教練（若由教練欄位建立則自動帶入）/ 會員 / 票券。
5. 建立完成後卡片自動收回。

## 三、查看／修改預約

**預設卡片僅顯示：時間、會員姓名。**

- **Hover**：Fish-eye 放大，顯示課程、教練等完整資訊。
- **Click**：原地展開完整編輯卡 —— 可直接修改課程/教練/票券/備註、可直接簽到、取消預約。

## 四、取消預約

取消按鈕直接於展開卡片底部顯示。點擊後於卡片內確認，**不使用 Modal**。

## 五、互動規則

- 同一時間僅允許一張卡片展開。
- 點擊其他卡片或空白處，自動收合上一張。
- 展開／收合動畫 **200–250ms**。
- 不使用 Modal、Drawer 或頁面跳轉。

## 六、設計優點

保持行事曆閱讀情境；操作步驟更少；不遮擋其他課程；高頻操作更快速；Hover 放大＋Click 展開形成一致互動語言。

## 使用者補充的視覺規則（2026-07-18 對話）

- **預設卡片內容標準化**：只顯示 **時間 + 會員姓名**（文字）；課程與教練改用**顏色**呈現：
  - 卡片**上方 20%** ＝ 課程顏色
  - 卡片**下方 80%** ＝ 教練顏色
- **移除「特殊卡」規則**：所有預約卡片規格統一標準化（取消團體/跑步機/溢位的特殊區與 `cal-ev-special` 樣式）。
- **週曆一次顯示整週 7 天**（已完成，版本 260718.0924）。
