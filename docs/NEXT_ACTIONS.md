# 下一步清單

更新日期：2026/05/25

這份文件只放最近最重要的下一步，避免 Codex 每次從長對話裡重新判斷優先順序。

## 最高優先

### 1. 完成 Supabase 資料同步

目的：

- 家裡與公司電腦可以共用同一份資料。
- 外部 Chrome、Codex 內建瀏覽器可以看到同一份資料。
- 未來員工端與會員端可以串接同一份資料。

需要：

- Supabase Project URL。
- Supabase anon public key。
- 執行 `supabase-schema.sql`。
- 將 `js/app.js` 的展示模式切換為 Supabase 模式。

### 2. 穩定預約行事曆

要持續檢查：

- 過去日期不可新增預約。
- 同一教練同一時段不可重複預約。
- 同時段一般預約最多 3 組。
- 小班團課獨立容量。
- 拖曳小卡失敗時只用右下角提示。
- 分期與續約章判定正確。

### 3. 穩定會員與票券資料

要持續檢查：

- 票券分批顯示，不要混在一起。
- 贈送堂數獨立顯示。
- 分期堂數與金額正確。
- 退費與儲值金邏輯正確。
- 新朋友完成教練課後可升級為會員。

## 中期目標

### 4. 員工手機端

目的：

- 教練可以看自己的課表。
- 教練可以協助簽到。
- 櫃台可以快速查看今日課程。

注意：

- 不要直接把後台塞進手機版。
- 手機版要另做簡化介面。

### 5. 會員端

目的：

- 會員可以查看自己的預約。
- 會員可以查看剩餘票券。
- 會員可以簽到或掃 QR。

注意：

- 會員端不能看到其他會員資料。
- 需要登入與權限控管。

## 先不要急著做

- 發票完整串接。
- 合約電子簽署。
- 複雜薪資獎金自動化。
- 太細的報表。
- 大型重構。

這些可以等預約、會員、票券與資料同步穩定後再做。
